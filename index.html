Web前端基础学习记录

发表于 2020-06-27 分类于 前端 阅读次数： Valine：

HTML部分

概述

<!doctype>
浏览器将按标准模式解析渲染页面。

<head></head>
设置网页文档的头部信息，一般不会显示浏览器窗口中。

<tittle></tittle>
设置网页文档的标题。显示在浏览器的标签栏或选项卡中。
给搜索引擎索引。

<meta>
定义文档的元数据。

常用标签

文本标签

<p align="对齐方式"> </p> /*align: left center right*/
默认隔行换行，段落有16px的上外边距和下外边距。

<br>或者<br />
单标签。用于换行。连续使用两个<br>将产生一个空行。

<h1></h1>
标题。共有6级。默认加粗且与下行产生一空行。

<strong></strong>
加粗，语气加重。该标签修饰的文本更容易吸引搜索引擎。

<em></em>
斜体。又类似<strong></strong>的作用，但语气较之稍弱。

&nbsp;
不换行键入空格。默认情况下在不同浏览器中显示的宽度有所不同。最好使用CSS样式属性text-indent来设置。

&实体名称;
输入特殊字符。

装饰性标签，设置文本的视觉差异。

文档结构标签

<header></header>
可以作为网页或者任何一块元素的头部信息。
不能嵌套<header> ``<footer>。

<article>独立内容</article>

<section id="块主题">块内容</section>
更多是从语义上对内容进行分块。

<aside>侧边栏内容</aside>

<footer>页脚内容</footer>

多媒体内容标签

<img src="图片文件路径" tittle="图片描述信息" alt="图片无法显示时的替换信息">

HTML 5 中的新标签。

列表标签

有序列表。

无序列表。

div标签

<div></div>
每个<div>标签独占一行，其宽度自动填满父元素宽度，并和相邻的块级元素依次垂直排列。

元素类型

block块元素：div p h1-h6 ul ol dt dd section header footer nav

inline行内元素：span a em strong

  • 不可设置height width
  • 不会独占一行
  • 不可以设置上下方向的外边距

inline-block行内块元素

  • 同一行
  • 可以设置height width
  • 可以设置4个方向的内外边距

超链接

<a href="目标端点" target="目标窗口名称">源端点</a>

href属性值

  • URL
    • http://
    • ftp:// 进入文件传输服务器
    • telnet:// 启动远程登录方式
    • news:// 启动新闻讨论组
    • mailto:
      <a href="mailto:邮件地址1?subject=content&cc=邮件地址2&bcc=邮件地址3">源端点</a>
  • javascript:code;
  • #

target属性值

  • _self（默认属性）
  • _blank：在新打开的窗口中打开链接文档
  • _parent：在上一级窗口中打开
  • _top：在浏览器的整个窗口中打开，忽略任何框架
  • 框架名称：在指定的浮动框架窗口中打开链接文档

基准URL标签

<base href="" target="">
用处是简化URL书写。在一个文档中最多出现一次，置于<head></head>标签对内

文件下载链接

<a href="file_URL">源端点</a>
目标文件后缀名是.doc .rar .zip .exe时，可以获得文件下载链接

书签链接

浮动框架

<iframe src="源文件地址" name="" width="" height="" frameborder=""></iframe>
作用：在页面中构建一个区域，以显示另一个HTML页面的内容。默认显示1px的边框。

表格

注意HTML5中很多原有属性不再支持。

<th></th>
表头单元格。默认居中对齐并加粗显示

表格分区。

表单

form标签

<form name="表单名称" method="提交方法" action="处理程序"></form>
method:表单数据从客户端传输到服务器的方法

  • get（默认）
    • 将表单内容附加到URL地址后，对提交信息长度有限制（不超过8KB）
    • 无保密性
    • 不能传送非ASCII码的字符
  • post
    • 将用户在表单中填写的数据包含在表单的主体中，一起传送给服务器上的处理程序
    • 包含ISO 10646中的所有字符

onsubmit:制定处理表单的脚本函数
enctype：设置MIME类型

input元素

text-文本框
<input type="元素类型" name="表单元素名称"/>
- maxlength
- size-长度（像素）
- value-默认值

password-密码框

hidden-隐藏域

  • 用于在不同页面中传递域中设置的值。

file-文件域

  • form的enctype需设置为“multipart/form-data

radio-单选框
<input type="radio" name="域名称" value="域值" checked[="checked"]/>

  • 同一组单选框的各个选项的name必须相同
  • checked表示此项默认被选中，一组单选框中最多只能有一个

checkbox-复选框

submit-提交按钮
<input type="submit" name="按钮名称" value="按钮显示文本"/>

button-按钮：激发表单动作
<input type="button" value="按钮显示文本" onclick="Javascript函数名" name="按钮名称"/>

reset-重置按钮
<input type="reset" name="按钮名称" value="按钮显示文本"/>

image-图像按钮：作用和提交按钮相同
<input type="image" name="按钮名称" src="图像路径" width="" height="" />

盒子模型


当我们指定一个 CSS 元素的width和height属性时，只是设置内容区域的宽度和高度。

盒子模型的属性

  • border-style/width/color
    • 如果有2个属性值：上下+左右
    • 如果有3个属性值：上-左右-下
    • 如果有4个属性值：顺时针方向
    • border-top|left|right|bottom-style
  • padding
  • margin

外边距的合并

相邻元素
当两个垂直外边距相遇时，它们将形成一个外边距。合并后的外边距的高度等于两个发生合并的外边距的高度中的较大者。

元素自身
无border无padding无content的空元素，上下边距分别和相邻元素合并，然后自己的上下边距合并。

父子元素
父元素中不设置padding和border时，父元素和子元素的外边距重合。当只设置子元素的margin-top，父子元素发生外边距合并，整体下移。

排版方式

标准流排版

不使用其他的与排列和定位相关的特殊CSS规则。
标准流中，块级元素独占一行，垂直放置；在水平方向自动伸展，直到包含它的父级元素的边界。行级元素在水平方向上一个接一个地排列。

浮动排版

大多数元素都在正常流中，要让一个元素不在正常流中，唯一的方法就是使之成为浮动元素或定位元素。
浮动的盒子在水平方向上移动，直到它的外边缘碰到包含框或另一个浮动框的边框为止。

特点

  • 浮动流中，无论是块级元素/行内元素/行内块级元素都可以水平排版，都可以设置宽高
  • 任何显示在浮动元素下方的HTML元素都会在网页中上移（好像浮动元素不存在一样），如果上移的元素中包含文字，则会环绕在浮动元素的周围（文字围移）
  • 向同一方向浮动的元素形成流式布局，排满一行或一行剩下的空间不够时，自动换行。如果前面已排列好的浮动元素的高度大于后面的浮动元素，换行时“卡住”。

浮动的副作用

  • 元素一旦设置为浮动，其下方的元素将上移，导致网页布局混乱
  • 当一个元素的所有子元素都设置为浮动时，由于脱离文档流，导致父元素的高度无法根据子元素自适应，收缩为一条线或高度仅为内边距+内容高度+上下边框（有瑕疵的解决方式：设置父元素高度，但仍不能自适应）
    • 在所有子元素的最后面添加一个div元素，并设置为clear：both;height:0px;；这个新加的元素有时反而可能造成网页布局混乱
    • 通过父元素的after伪元素清除浮动
